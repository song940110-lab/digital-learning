Chapter 01

OpenAI API

學習如何申請與使用 OpenAI API，設計能生成會議記錄的 Prompt，並了解如何控制 API 費用。

📖 4 個小節 ⏱ 約 30 分鐘 🎯 含實作練習

1什麼是 OpenAI API？

OpenAI API 是 OpenAI 公司提供的一個程式介面，讓開發者可以透過程式碼直接使用 GPT-4、Whisper 等 AI 模型的能力。你不需要自己訓練 AI，只需要傳送文字請求，AI 就會回傳結果。

💡
用一個比喻來說：OpenAI API 就像一個聰明的助理，你用文字告訴它「請把這段逐字稿整理成會議記錄」，他就會幫你完成並把結果回傳給你。

2申請 API Key

API Key 是你使用 OpenAI API 的「身份證」。每一次呼叫 API 時都需要帶上它，OpenAI 才知道是誰在使用，並計算費用。

⚠️
API Key 要保密！不可以把 API Key 放在公開的程式碼或 GitHub 上，否則別人可以用你的額度消費。

步驟說明

1

前往 OpenAI 官網

開啟瀏覽器，前往 platform.openai.com，點右上角 Sign Up 建立帳號，或用 Google 帳號登入。

2

進入 API Keys 頁面

登入後，點左側選單的 API Keys，或直接前往 platform.openai.com/api-keys。

3

建立新的 API Key

點 + Create new secret key，輸入一個名稱（例如 MeetingDoc-Key），按下 Create secret key。

4

複製並保存

Key 只會顯示一次，請立即複製並貼到安全的地方（例如記事本）。格式通常是 sk-proj-xxxxxxxx...。

5

儲存在環境變數（推薦）

在 Windows 上，不要把 Key 直接寫在程式碼裡。使用環境變數或 .env 檔案來存放：

設定環境變數（Windows PowerShell） 
[System.Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "sk-proj-你的key", "User")
在 C# 中讀取
var apiKey = Environment.GetEnvironmentVariable("OPENAI_API_KEY");

3呼叫方式與參數

OpenAI API 的呼叫方式是傳送一個 JSON 格式的請求到指定網址，API 就會回傳 AI 的回答。在 C# 裡，我們使用官方的 OpenAI NuGet 套件來簡化這個過程。

基本呼叫結構

C# — 基本 API 呼叫範例 
using OpenAI;
using OpenAI.Chat;

// 1. 建立 client
var client = new ChatClient(
    model: "gpt-4o-mini",
    apiKey: Environment.GetEnvironmentVariable("OPENAI_API_KEY")
);

// 2. 準備訊息
var messages = new[] {
    new SystemChatMessage("你是一個專業的會議記錄整理助理。"),
    new UserChatMessage(transcript)
};

// 3. 發送請求並取得結果
ChatCompletion result = await client.CompleteChatAsync(messages);
string meetingMinutes = result.Content[0].Text;

重要參數說明

model
要使用的 AI 模型，建議用 gpt-4o-mini
SystemChatMessage
設定 AI 的角色與行為規則，每次對話開頭都要帶
UserChatMessage
使用者送出的訊息，就是你要讓 AI 處理的逐字稿內容
MaxOutputTokenCount
限制回應的最大字數（token），避免回應過長或費用過高

加上 MaxOutputTokenCount

C# — 加上 token 限制 
var options = new ChatCompletionOptions
{
    MaxOutputTokenCount = 2000  // 約 1500 個中文字
};

ChatCompletion result = await client.CompleteChatAsync(messages, options);

4Prompt 設計技巧

Prompt 就是你給 AI 的「指令」。Prompt 寫得好，AI 回傳的會議記錄就越符合你的需求。這裡介紹針對會議記錄場景的設計技巧。

基本結構：System + User 分開

📌
System Message 負責設定 AI 的角色與輸出格式規則。
User Message 負責放入實際要處理的逐字稿內容。
這樣分開寫，格式規則不用每次重複，維護也比較方便。

實際範例

❌ 不好的 Prompt
請幫我整理會議記錄：[逐字稿內容]

問題：沒有規定格式，AI 每次輸出都不一樣，無法穩定對接 Word 模板。

✅ 好的 Prompt（System Message） 
你是一個專業的會議記錄整理助理。
請根據以下逐字稿，整理成結構化會議記錄。

輸出格式如下（請嚴格遵守）：
【會議主題】：xxx
【日期時間】：xxx
【出席人員】：xxx
【討論事項】：
1. xxx
【結論與決議】：
1. xxx
【待辦事項】：
| 負責人 | 項目 | 截止日 |
|--------|------|--------|

注意事項：
- 若資訊不明確，請寫「未提及」
- 不要加入逐字稿沒有的內容
- 中文輸出

優點：固定格式、方便後續用 C# 解析並填入 Word 模板。

進階技巧

🎯

指定語言

加上「請用繁體中文輸出」，避免 AI 混用簡體或英文。

📏

控制長度

加上「每個討論事項摘要不超過 50 字」，讓輸出長度一致。

🔒

禁止幻覺

加上「不要加入逐字稿沒有的內容」，防止 AI 自己捏造資訊。

🔁

反覆測試

用多份不同類型的逐字稿測試，找出 Prompt 不穩定的地方再調整。

5費用與用量控制

OpenAI API 是按照 token 計費的。Token 是 AI 處理文字的最小單位，大致上：1 個英文單字 ≈ 1 token；1 個中文字 ≈ 1～2 token。

gpt-4o-mini 費用參考

類型 費用（每 1M tokens） 換算
輸入（你送出的文字） $0.15 USD 100萬字約 NT$5
輸出（AI 回傳的文字） $0.60 USD 100萬字約 NT$20
💰
一場 1 小時的會議，逐字稿約 5,000～8,000 字。用 gpt-4o-mini 處理一次的費用大約不到 NT$1。

設定用量上限（推薦）

1

設定每月消費上限

前往 platform.openai.com 的 Limits 頁面，設定 Monthly budget limit，例如 $5 USD，超過就自動停止。

2

在程式碼限制 token 數

設定 MaxOutputTokenCount 避免單次請求費用爆炸： 

MaxOutputTokenCount = 2000  // 約 1500 個中文字，足夠一般會議
3

查看用量

前往 platform.openai.com/usage 可以看到每天、每個 Key 的使用量。
